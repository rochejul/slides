Overview

What are CSS4 variables?

They are a nice way to include true a programmative variable which includes a value

A first notion of "CSS variables" where included with CSS preprocessors like LESS, Stylus, SASS

A "CSS variable" in LESS

                @nice-blue: #5B83AD;
@light-blue: @nice-blue + #111;

#header {
  color: @light-blue;
}

What are the benefits?

  • A cleaner code (moreover with semantic name on variables)
  • A most reusable code (duplicating values versus using value)
  • Increase update velocity
  • Increase bug resolution velocity

Where to declare my CSS variable?

You have to declare it into a "CSS rule" block

                .block {
   /* CSS Rule block */
}

How to declare my CSS variable?

Like a CSS property, prefixing with --

                .block {
   --color: blue
}

WARNING

Like a CSS property, we have to specify a valid CSS property value

                :root {
 --size: 20;
 --other-size: 20px;
}

div {
  font-size: var(--size)px; /*WRONG*/
  line-height: var(--other-side); /*GOOD*/
}

Scoping?

Like JavaScript, CSS variables have a scoping. We could have:

  • Local scope
  • Global scope

Locale scope

Only for the div elements

                .block {
   --color: blue
}

Global scope

For all elements

                :root {
   --color: blue
}

How to use my CSS variable?

A new CSS method is available, var

                :root {
  --color: red;
  --font: 'Arial';
}

.block {
   color: var(--color);
   font-family: var(--font);
}

Hold on, var has got two parameters!

The second argument is a "default value"

The CSS engine will use it if the variable cannot be found, or the associated value is not a valid one

                .component .header {
  color: var(--header-color, blue);
}

.component .text {
  color: var(--text-color, black);
}

.component {
  --text-color: #080;
}

One more thing

We could use CSS variables directly into the style attribute!

                <section style="--color: red"></section>

Further details on scoping

One question

If we have a local variable and a global variable with the same name, who wins?

This is the last local variable (like JavaScript)

                :root {
  /* Global scope */
  --color: red;
  --font: 'Arial';
}

.block {
   /* --color === blue, --font === 'Arial' */
   --color: blue;
}

Sorry, I have lied :(

On a previous slide, I said that a local variable is limited to the CSS rule block. That's not fairly true

Like other CSS properties, CSS variables applied the CSS logic, which are:

  • Inheritance of CSS rules
  • Cascading of CSS rules

So, we could have

                div {
  --color: red;
}

div.test {
  color: var(--color)
}

div.ew {
  color: var(--color)
}

And we have a new scope when using media queries or supports

                :root {
   --color: red;
}

@media screen and (min-width: 600px) {
   --color: green;
}

@supports (display: grid) {
   --color: blue;
}

JavaScript API

Update of CSSStyleDeclaration Object

Each dom element has got a style property

This property is a type of CSSStyleDeclaration

CSSStyleDeclaration allows to modify easily the style of your DOM element

Basic example

For futher details, go to the developer mozilla page 

                var declaration = document.styleSheets[0].rules[0].style;
declaration.setProperty('margin', '1px 2px');
// Equivalent to:
// declaration.margin = '1px 2px';

So, basically ...

CSSStyleDeclaration exposes a setProperty to modify a value

CSSStyleDeclaration exposes a getProperty to retrieve a value

We can use it for CSS variables!

To read / update a CSS variable of the current node element

                const root = document.documentElement
const element = document.querySelector('.my-element');

root.style.setProperty('--bg', 'black');
root.style.setProperty('--bg-text', 'grey');
element.style.setProperty('--bg-text', 'white');
