The ugly parts

The f***ng jumping line

You want to align two blocks on the same line...

                <div class="my-element my-element--left">Left element</div>
<div class="my-element my-element--right">Right element</div>

..., without border, margin, padding

                .my-element {
  border: none;
  display: inline-block;
  height: 50px;
  margin: 0 0 0 0;
  padding: 0 0 0 0;
}

.my-element--left {
  background: orange;
  width: 45%;
}

.my-element--right {
  background: purple;
  width: 55%;
}

Results

The reason?

"Elements" between inline-blocks should have a minimal width. If not specified, the browser applied one (between 2 or 3 pixels)

In our case? The carriage return or the space between our tags ...

Change to

            <div class="my-element my-element--left">Left element</div><div class="my-element my-element--right">Right element</div>

Tada!

Height 100% may not do what you think it does

We want our element take the 100% height of the zone

                .my-element {
   background-color: orange;
   color: white;
   height: 100%;
}

But we do not have the expected behavior

The reason?

The purcentage css unit is based on a context

That means height 100% means we want 100% of the parent height, not the page

Ok, we try to fix it

                html,
body {
   height: 100%;
}

.my-element {
   background-color: orange;
   color: white;
   height: 100%;
}

Tada! (with overlap due to parent padding / margin)

Opacity changes the z-index

Talk is cheap, let's play with it

This example is based on an article by Philip Walton

  • Link: https://philipwalton.com/articles/what-no-one-told-you-about-z-index/

What Philip Walton says

  • Basically, elements with a higher z-index are stacked in front of elements with a lower z-index
  • BUT when you introduce the position property into the mix, any positioned elements (and their children) are displayed in front of any non-positioned elements

Some stacking contexts are created...

  • ...when an element is the root of a document
  • ...when an element has a position which is not "static" or "relative"
  • ...when an element has got a values from properties like opacity, transform, filter

The bad parts

Vertical padding is relative to element’s width

What does it mean?

So padding-top: 50% does not add 50% of the original height of the element as padding, but 50% of the width of the parent element

Results

Margins overlap, but only sometimes

Cases of margins adjoining

  • Sibling elements: The bottom margin of an element collapses with the top margin of its proceeding sibling.
  • Child elements:
    1. The top margin of an element collapses with the top margin of its first child element.
    2. The bottom margin of an element collapses with the bottom margin of its last child element.
  • No size elements: The top and bottom margins of an element collapse if the element has no height, padding, or border.

The good parts

The power of CSS units

Do you know we use only a few units?

  • px for pixels
  • % for purcentage
  • em for element measure
    1. Measuring the width of printed matter, equal to the height of the type size being used
    2. Dedicated for font sizes

But we have plenty mores!!

Basics sizing

  • cm for centimeters
  • mm for millimeters
  • in for inches
  • pt for points (1/72 of a inch)
  • pc for picas (12 points)

Font sizes

  • rem for root em (no more context sizing!)
  • ex for height of the lowercase x
  • ch for the width of the 0

Be carefull when to use it

Recommanded Sometimes Never
Screen rem, em, px, % ex pt, cm, mm, in, pc
Printer rem, em, cm, mm, in, pt, pc, % px, ex

Some units for transformation

  • deg for degrees
    1. Especially for rotation

Some units based on the viewport size!

  • vw for 1/100th of the screen width
  • vh for 1/100th of the screen height
  • vmin for 1/100th of the smallest screen side
  • vmax for 1/100th of the largest screen side

Intrinsic And Extrinsic Sizing

Boxes on your webpage have a size, even if you haven’t given them one

See the sizing spec

Max content

Per default, a box will always try to take the max content size

However, you could specify that it should take instead the min content or a fit content

  • max-content: the maximal size of the content of the boxes
  • min-content: the minimal size of the content of the boxes

Example

            .min-content {
  width: min-content;
}

.max-content {
  width: max-content;
}

at CSS rules

What is an at rule?

The at-rule is a statement that provides CSS with instructions to perform or how to behave.

Always have the following syntax: @[KEYWORD] (RULE);

@charset

Define the charset of your file

Place it to the top of the file

            @charset "UTF-8";

@import

To import css files between them

Note: in production mode, we will have an http request for each import: try to contatenate instead

            @import 'global.css';

@namespace

This rule is particularly useful for applying CSS to XML HTML (XHTML) so that XHTML elements can be used as selectors in the CSS.

Few use cases now

            /* Namespace for XHTML */
@namespace url(http://www.w3.org/1999/xhtml);

@font-face

Declare your own font!

            @font-face {
  font-family: "Nunito";
  font-style: normal;
  font-weight: 200;
  src: url("./nunito/nunito-v9-latin-200.eot"); /* IE9 Compat Modes */
  src: local("Nunito ExtraLight"), local("Nunito-ExtraLight"),
    url("./nunito/nunito-v9-latin-200.eot?#iefix") format("embedded-opentype"), /* IE6-IE8 */ url("./nunito/nunito-v9-latin-200.woff2") format("woff2"), /* Super Modern Browsers */ url("./nunito/nunito-v9-latin-200.woff") format("woff"), /* Modern Browsers */ url("./nunito/nunito-v9-latin-200.ttf") format("truetype"), /* Safari, Android, iOS */ url("./nunito/nunito-v9-latin-200.svg#Nunito") format("svg"); /* Legacy iOS */
}

@media

Define query condition to apply some style

            @media only screen
  and (min-device-width: 320px)
  and (max-device-width: 480px)
  and (-webkit-min-device-pixel-ratio: 2) {

    .module { width: 100%; }

}

@keyframes

Define a scenario of an animation

            @keyframes pulse {
  0% {
    opacity: 1;
  }
  100% {
    opacity: 0.2;
  }
}

.my-element {
  animation: pulse linear 1s infinite alternate;
}

@page

Usefull when media is "print" mode

Related pseudo-elements:

  • :first (when first page)
  • :blank (when the page is blank)
  • :left (when print left to right)
  • :right (when print right to left)

Check if a CSS feature is available

Optimize painting

The CSS painting contains three steps

  • Layout
  • Paint
  • Composite

When you change a CSS properties value, the browser needs to react to your change

A change in width requires the browser

  • To update layout
  • Then "paint" any pixels that have changed
  • Then "composite" them together

Some CSS properties can be changed more inexpensively

A good site to identify them: https://csstriggers.com/

Mastering these changes helps to master

  • Repaints (occurs when changes are made to elements that affect visibility but not the layout: Ex: opacity, background-color)
  • Reflows (refers to the re-calculation of positions and dimensions of all elements)

Did yo know there is a CSS property to help you?

will-change CSS property

Optimizes animations by letting the browser know which properties and elements are just about to be manipulated, potentially increasing the performance of that particular operation.

Possible values

  • auto: standard browser optimizations will be applied
  • scroll-position: indicates that the element’s scroll position will be animated sometime in the near future
  • contents: the contents of an element is expected to change so the browser will not cache this element’s content
  • Any css property: any user-defined property such as transform or opacity that we want will-change to be applied to.

And there is another one

contain CSS property

Limit the scope of the browser's styles, layout and paint work.

Possible values

  • none
  • Combination of layout style paint size
    • layout: ensures that the containing element is totally opaque for layout purposes
    • paint: ensures that the descendants of the containing element don’t display outside its bounds
    • size: ensures that the containing element can be laid out without needing to examine its descendants
    • style: ensures that, for properties which can have effects on more than just an element and its descendants
  • strict: combination of layout style paint size
  • content: combination of layout style paint
